# K² v3 — Свёрточный AI-прекондиционер для ADDA

Полный гайд по обучению, экспорту и использованию.

## Что это такое

**Прекондиционер** — вспомогательная операция, ускоряющая итерационный решатель BiCGStab при решении системы линейных уравнений в методе дискретных диполей (DDA). Без прекондиционера — тысячи итераций. С хорошим — десятки.

**K² v3** — нейросеть (13.7M параметров), которая предсказывает **свёрточное ядро** прекондиционера. Ядро задаёт для каждого смещения (dx, dy, dz) в радиусе 7 клеток блок 3×3 комплексных чисел. K² означает возведение ядра в квадрат в частотной области (`M_hat = K_hat @ K_hat`), что удваивает эффективный радиус с 7 до 14 клеток без дополнительных параметров.

Модель **универсальная** — один файл весов (53 MB) для любых форм частиц и параметров.

## Архитектура

### Входы
- **Физические параметры**: m_re (показатель преломления), m_im (мнимая часть), kd (параметр размера = 2π·d/λ), log(grid)
- **Occupancy grid**: бинарная 3D сетка (0/1), описывающая форму частицы

### Модель
```
Occupancy grid (1, 1, G, G, G)
    ↓
ShapeEncoder3D (3D свёрточная нейросеть)
  Conv3d(1→16) + ReLU + MaxPool
  Conv3d(16→32) + ReLU + MaxPool
  Conv3d(32→64) + ReLU + MaxPool
  AdaptiveAvgPool3d(1) + Linear → 16 чисел (вектор формы)
    ↓
Склейка: [m_re, m_im, kd, log(grid), shape_embed] = 20 чисел
    ↓
MLP: 20 → 512 → ReLU → 512 → ReLU → 512 → ReLU → 25 542
    ↓
Ядро свёртки K: 1419 точек × 3 × 3 complex  (r_cut=7)
    ↓
FFT → K_hat @ K_hat → M_hat  (K² = ядро в квадрате, eff. r_cut=14)
```

### Размер модели
- ShapeEncoder3D: 70 704 параметра
- MLP: 13 639 110 параметров
- Итого: **13 709 814**

## Обучение

### Команда
```bash
python train_v7/train.py \
    --name universal_r7_k2_v3 \
    --device 0 --save \
    --loss adversarial \
    --squared_kernel \
    --r_cut 7 \
    --hidden_size 512 \
    --num_layers 4 \
    --grid_min 8 --grid_max 64 \
    --curriculum_frac 0.2 \
    --lr 5e-4 \
    --ema_decay 0.999 \
    --warmup_steps 500 \
    --num_steps 60000
```

### Параметры обучения

| Параметр | Значение | Что делает |
|----------|----------|-----------|
| `--squared_kernel` | — | Включает K²: M_hat = K_hat @ K_hat |
| `--r_cut 7` | 7 | Радиус стенсиля (1419 точек в сфере радиуса 7) |
| `--hidden_size 512` | 512 | Ширина MLP |
| `--num_layers 4` | 4 | Число слоёв MLP |
| `--loss adversarial` | — | Adversarial probe loss (ищет наихудший вектор и оптимизирует на нём) |
| `--grid_min/max 8/64` | 8–64 | Диапазон размеров сетки при обучении |
| `--curriculum_frac 0.2` | 0.2 | Первые 20% обучения: grid постепенно растёт от 8 до 64 |
| `--lr 5e-4` | 0.0005 | Скорость обучения (AdamW) |
| `--ema_decay 0.999` | 0.999 | Скользящее среднее весов (сглаживает шум, EMA модель обычно лучше) |
| `--num_steps 60000` | 60000 | Число шагов обучения |

### Процесс обучения (на каждом шаге)

1. **Генерация случайной задачи**: m ∈ [1.5, 4.0], kd ∈ [0.2, 0.8], grid ∈ [8, max], случайная форма
2. **Построение FFTMatVec**: A·v через FFT (матрица DDA задачи)
3. **Forward**: occupancy grid + параметры → стенсильное ядро K
4. **K²**: FFT(K) → K_hat @ K_hat → M_hat
5. **Adversarial probe loss**: найти худший вектор z (10 итераций power iteration), loss = ||M·A·z − z||²/||z||²
6. **Backward + AdamW + EMA**: обновление весов

### Время обучения
~8 часов на RTX 4090 (24 GB). На маленьких grid — 30+ steps/sec, на больших — 1-2 steps/sec.

### Мониторинг
```bash
tail -f results/universal_r7_k2_v3/train.log
# loss 0.7-1.0 — хорошо
# "New best (EMA): speedup X.XXx" — ключевая метрика
```

## Экспорт для ADDA

### Команда
```bash
python apps/export_universal_precond.py \
    --checkpoint models/k2v3/checkpoints/best_model.pt \
    --squared_kernel \
    --shape sphere \
    --grid 33 \
    --m_re 3.0 --m_im 0.0 \
    --kd 0.4189 \
    --output /tmp/precond.precond
```

### Параметры экспорта
- `--squared_kernel` — обязательно для K² v3
- `--shape` — форма: sphere, cube, ellipsoid, cylinder, capsule
- `--grid` — размер сетки (должен совпадать с ADDA)
- `--m_re`, `--m_im` — показатель преломления
- `--kd` — параметр размера = 2π/dpl. При dpl=15: kd = 0.4189
- `--ay`, `--az` — пропорции осей эллипсоида (если shape=ellipsoid)

### Что делает скрипт
1. Создаёт позиции диполей для формы
2. Строит occupancy grid
3. Прогоняет нейросеть → получает ядро K (1419 × 3 × 3 complex)
4. Вычисляет K² в частотной области → получает расширенный стенсиль (~10649 точек)
5. Записывает стенсиль в .precond файл (mode=3 CONVSAI)

### Формат .precond

Бинарный файл, ~1.6 MB:
```
Header (40 байт): magic, n, n_stencil, mode=3, reserved
Data:
  stencil: n_stencil × 3 × int32     (смещения dx, dy, dz)
  kernel:  n_stencil × 18 × float64  (3×3 complex блоки, re/im чередуются)
```

## Запуск ADDA

```bash
LD_LIBRARY_PATH=$HOME/.local/lib \
adda/src/seq/adda \
    -grid 33 \
    -m 3.0 0.0 \
    -shape sphere \
    -iter bicgstab \
    -eps 5 \
    -dpl 15 \
    -precond /tmp/precond.precond
```

**Важно:** использовать `-grid N -dpl 15`, не `-size X -dpl 15`.

## Преимущества

- **Один файл весов** для всех задач — не нужен переэкспорт при смене m или grid
- **Компактный стенсиль**: ~10K записей, файл ~1.6 MB
- **Быстрое применение**: стоимость = 1 FFT-свёртка ≈ 1 умножение A·v
- **Проверен** на сотнях конфигураций в ADDA

## Ограничения

- Эффективный радиус 14 клеток — на grid > 48 эффективность снижается
- 13.7M параметров (vs 228K у Spectral)
- На g48 m=3.0 не сходится (Spectral справляется)
